# Competitor Landscape: Autonomous QA Agents

## The Rise of Agentic QA (2025-2026)
The market is shifting from "AI Assistants" (generating a Selenium script) to "AI Agents" (reasoning about a UI and autonomously taking action).

## Key Players
1. **Mabl:** Heavyweight incumbent leaning into "Agentic Workflows." Uses plain English to autonomously execute tests and perform root cause analysis.
2. **Momentic:** Uses a reasoning engine to execute goals in natural language, actively handling edge cases on the fly.
3. **DeepEval & Ragas:** Developer-first, open-source evaluation frameworks. DeepEval applies unit testing principles directly to LLM agents.
4. **Spur:** Vision-based agents that interact with elements based on intent, entirely bypassing brittle DOM selectors.
5. **testers.ai:** Spun out of ex-Google Chrome testing teams. Autonomous agents write and run the tests.
6. **LambdaTest KaneAI:** LLMs for natural language test creation and debugging.

## Core Advancements
- **Self-Healing:** Machine learning adapting to UI changes and mutating the test scripts automatically.
- **Drift Detection:** Monitoring production for performance or visual degradation.
- **Visual AI:** Catching subtle layout shifts via computer vision (Applitools).

## Actionable Phantom Takeaway
The QA automation space is crowded with "Self-Healing Test" tools (Mabl, LambdaTest). **Phantom must avoid competing in the pure "Functional QA Automation" bucket.** We must strictly target the "UX/UI Empathy and Friction" bucket.
